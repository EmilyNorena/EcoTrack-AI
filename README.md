# EcoTrack-AI

## Master Prompt
```
Create a simple React web app called EcoTrack AI.

The app should allow small businesses to type their daily activities in natural language (example: "We used 5 delivery vans and consumed 200 kWh of electricity").

The system should analyze the text and display a simple carbon footprint estimate.

Use a clean green sustainability design.
Use Tailwind if possible.
```

We use de IDE Cursor used before in the course.

## Challenges

During the project's deployment to GitHub, a synchronization error occurred between the local and remote branches. When running `git push`, Git rejected the changes because the remote branch had a previous history, and when using `git pull`, another error appeared related to a lack of tracking between branches. To resolve this, AI was used for debugging support: the exact terminal message was shared with it, and based on this, the AI ​​explained the cause of the conflict and proposed the necessary commands to link the local and remote branches and complete the integration. This allowed the issue to be resolved without writing manual code and demonstrated the value of AI as a technical co-pilot within the development process.

## How Vibe Coding accelerated the process

The Vibe Coding methodology accelerated the development of the MVP because it reduced the need to program every detail manually and allowed working from natural language instructions. Instead of spending time building the interface, core logic, and bug fixes from scratch, it was possible to leverage AI tools to generate components, refine the design, guide functional implementation, and quickly resolve technical roadblocks. Compared to a traditional approach, this process streamlined iteration, facilitated experimentation, and allowed for a greater focus on the product vision and user experience rather than on code syntax.
